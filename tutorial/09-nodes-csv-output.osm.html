09 - Generating CSV output

Run

OverpassQL can also be used to generate simple CSV exports.

This is achieved by using the [out:csv] query setting.

In order for the API to know how to construct the CSV output you must provide a specification of what fields (columns) to include. These fields are either:

  • a special field name, which is predefined by the API, e.g. ::id
  • the name of a tag

The special field names include:

  • ::id - the database id of the object
  • ::type - its type, e.g. node, way or relation
  • ::lat - its latitude
  • ::lon - its longitude

The list of columns to include are specified as parameters in the query setting.

Its also possible to customise whether a header line is included in the CSV file and the column separator to use. The default is to include a head and use tabs.

The general syntax for the out:csv setting is:

[out:csv (columns; header; col-separator)]

Where:

  • columns is a comma separated list of names
  • header is either true or false
  • col-separator is the desired separator provided as a string.

The following query generates a simple comma-separated table containing the id, coordinates and names of caves. But some variations are also provided.

Note: if you want to include tags in your CSV output, then the out keyword of your query MUST generate tags. E.g out body or out meta.

Note: The latitude and longitude values only really apply to nodes. Ways and relations are more complex spatial objects. However you can use the out keyword to automatically generate a single location ("centroid") for those types of feature. Specifying out body center will do the trick if your query returns ways or relations.

Tip: when generating CSV output, if there is a query error or timeout then you will get empty results. I'd recommend writing and testing your basic query first, so you're sure its working. And then customise it further to generate the desired CSV output.
